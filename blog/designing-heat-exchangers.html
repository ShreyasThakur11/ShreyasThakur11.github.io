Designing Heat Exchangers in the Browser

Published on Feb 20, 2026 â€¢ 5 min read

Heat exchanger design is typically the domain of expensive software like HTRI or Aspen EDR. These tools solve complex differential equations to ensure that a unit meets thermal duty requirements within allowable pressure drops. For my portfolio, I wanted to demonstrate that these fundamental chemical engineering principles could be implemented in a lightweight, web-based tool using JavaScript.

The Challenge: Iterative Solving

The core challenge in exchanger design is that many variables are interdependent. For example, the heat transfer coefficient ($U$) depends on the fluid velocity, which depends on the tube count, which depends on the required area... which depends on $U$.

To solve this circular dependency, we use an iterative approach:

  • Assume an initial Overall Heat Transfer Coefficient ($U_{assumed}$).
  • Calculate the required Area ($A = Q / (U \cdot LMTD)$).
  • Select a standard geometry (Tube count, Shell diameter) that fits this Area.
  • Rate the selected geometry to find the actual coefficients ($h_i, h_o$) and pressure drops.
  • Compare $U_{calc}$ vs $U_{assumed}$. If they differ, update the assumption and repeat.

Implementing the Kern Method

For the Shell & Tube Design Tool, I implemented the Kern Method. This provides analytical equations for the shell-side heat transfer coefficient ($h_o$) based on baffle spacing and shell diameter.

// Simplified Shell-Side Calc (Kern) function getShellSideH(flow, properties, geometry) { const { De, B, Ds, Pt } = geometry; // Equivalent Dia, Baffle Spacing, Shell Dia, Pitch // Crossflow Area const as = (Ds * B * (Pt - Do) / Pt) / 144; // standard Kern eq const Gs = flow / as; // Mass Velocity // Reynolds Number const Re = (De * Gs) / mu; // Nusselt (jH factor lookup approximated) const jH = 0.5 * Math.pow(Re, 0.55); const ho = jH * (k/De) * Math.pow(Pr, 0.33); return ho; }

Why Client-Side?

By building this in JavaScript, the tool is strictly "Serverless". Benefits include:

  • Zero Latency: Calculations happen instantly as you type.
  • Privacy: No proprietary process data is sent to a cloud server.
  • Accessibility: It runs on any device with a browser, even offline.

Next Steps

While the current tool handles single-phase liquid-liquid duties, future updates could include:

  • Phase change (Condensers/Reboilers).
  • Vibration analysis to prevent tube failure.
  • Cost estimation based on weight and material.
